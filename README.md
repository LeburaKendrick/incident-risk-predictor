# incident-risk-predictor

> Flask REST API that uses a machine learning classifier to predict site incident risk from environmental and operational conditions — with AI-generated safety briefings via the Claude API.

## Overview

`incident-risk-predictor` exposes a `/predict` endpoint that accepts live site conditions — wind speed, temperature, working height, crew size, logged hazards, and time since last inspection — and returns:

1. A **risk classification** (low / high) from a trained Random Forest model
2. A **probability score** for the high-risk outcome
3. An **AI-written safety briefing** generated by Claude, referencing the specific conditions and the relevant ISO 45001 clause

The model is trained on labelled historical site data. In production, this would be replaced with a larger dataset from actual HSE records.

---

## Features

- `POST /predict` — run risk inference on site conditions
- `GET /health` — service status and model metadata
- Random Forest classifier with probability outputs
- Claude API integration for human-readable risk summaries
- ISO 45001 clause referencing in generated briefings
- Clean JSON responses, ready to plug into any frontend or dashboard

## Getting Started

### Prerequisites

- Python 3.9+
- An Anthropic API key ([get one here](https://console.anthropic.com))

### Installation

```bash
git clone https://github.com/your-handle/incident-risk-predictor.git
cd incident-risk-predictor
pip install -r requirements.txt
```

### Set environment variable

```bash
export ANTHROPIC_API_KEY=your_key_here
```

### Run

```bash
python app.py
```

API will be live at `http://localhost:5000`

---

## Usage

### Predict risk for a site

```bash
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "wind_speed": 35,
    "temp_celsius": 3,
    "height_metres": 25,
    "crew_size": 8,
    "hazards_logged": 6,
    "hours_since_inspection": 36
  }'
```

### Example response

```json
{
  "risk_level": "high",
  "risk_probability": 0.87,
  "ai_summary": "This site presents a HIGH risk profile, primarily driven by the combination of elevated wind speeds (35 km/h) and a 36-hour gap since last inspection. The supervisor should immediately suspend work at height and conduct a rapid site walkdown before resuming operations. ISO 45001:2018 Clause 8.1.3 (Management of change) and Clause 8.2 (Emergency preparedness) are directly applicable.",
  "iso_reference": "ISO 45001:2018",
  "recommendation": "Halt operations pending review"
}
```

### Health check

```bash
curl http://localhost:5000/health
```

## Input Features

| Field | Type | Description | Unit |
|---|---|---|---|
| `wind_speed` | float | Current wind speed on site | km/h |
| `temp_celsius` | float | Ambient temperature | °C |
| `height_metres` | float | Maximum working height | metres |
| `crew_size` | int | Number of workers on site | — |
| `hazards_logged` | int | Hazards recorded today | count |
| `hours_since_inspection` | float | Time since last formal inspection | hours |

## Extending the Model

To train on real data, replace the `X_train` / `y_train` arrays with a CSV load:

```python
import pandas as pd
df = pd.read_csv("site_data.csv")
X_train = df[["wind_speed", "temp_celsius", "height_metres",
              "crew_size", "hazards_logged", "hours_since_inspection"]].values
y_train = df["incident_occurred"].values
```
